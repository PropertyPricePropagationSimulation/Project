# EstateFlow 화면 설계서

## 1. 문서 개요

본 문서는 EstateFlow 프론트엔드 구현 화면을 기준으로 작성한 화면 설계서이다.  
Vue Router의 라우팅 구조, 화면 컴포넌트, API 모듈, 인증 정책을 기준으로 화면 목적과 사용자 액션을 정리한다.

## 2. 서비스 화면 구성

| 구분 | 화면명 | 경로 | 접근 권한 | 주요 목적 |
|---|---|---|---|---|
| 메인 | 홈 | `/` | 전체 | 서비스 진입, 주요 기능 이동 |
| 분석 | 부동산 충격 전파 분석 | `/analysis` | 전체 | 이벤트 기반 지역별 가격/거래량 변화 시각화 |
| 거래 | 아파트 거래 검색 | `/search` | 전체 | 지역/연월 조건 기반 실거래 조회 |
| 인증 | 로그인 | `/login` | 전체 | JWT 토큰 발급 및 로그인 상태 전환 |
| 인증 | 회원가입 | `/register` | 전체 | 신규 회원 등록 |
| 회원 | 마이페이지 | `/mypage` | 로그인 필요 | 회원 정보 관리, 생성 리포트/시나리오 확인 |
| 공지사항 | 공지사항 목록 | `/notices` | 전체 | 공지 목록 조회 및 페이지 이동 |
| 공지사항 | 공지사항 상세 | `/notices/:id` | 전체 | 공지 상세 내용 조회 |
| 공지사항 | 공지사항 작성 | `/notices/write` | 로그인 필요 | 공지 작성 |
| Q&A | Q&A 목록 | `/qna` | 전체 | 질문 목록 조회 및 작성 진입 |
| Q&A | Q&A 상세 | `/qna/:id` | 전체 | 질문 상세, 댓글 조회/작성 |
| Q&A | Q&A 작성 | `/qna/write` | 로그인 필요 | 질문 등록 |
| 시나리오 | 시나리오 상세 | `/scenarios/:scenarioId` | 로그인 필요 | 분석 기반 참여자 시나리오와 AI 해설 확인 |

## 3. 공통 화면 정책

### 3.1 공통 레이아웃

| 항목 | 설계 내용 |
|---|---|
| Header | 로고, 분석, 거래 검색, 공지사항, Q&A 메뉴 제공 |
| 인증 영역 | 비로그인 시 로그인/회원가입 버튼, 로그인 시 닉네임/마이페이지/로그아웃 제공 |
| Footer | 일반 페이지 하단 공통 정보 표시 |
| API Base URL | 모든 프론트 API는 `/api` 기준으로 호출 |
| 인증 헤더 | `http` 인스턴스 요청 시 `localStorage.access_token`을 `Authorization: Bearer`로 자동 첨부 |
| 토큰 갱신 | 401 응답 시 refresh token으로 access token 재발급 후 요청 재시도 |
| 로그인 가드 | `meta.requiresAuth`가 있는 라우트 접근 시 미로그인 사용자는 `/login?redirect=...`로 이동 |

### 3.2 상태 처리

| 상태 | 처리 방식 |
|---|---|
| 로딩 | 화면별 `loading`, `submitting`, `reportStore.loading`, `scenarioStore.loading` 상태로 버튼 비활성화 또는 로딩 문구 표시 |
| API 오류 | `parseApiError` 또는 화면별 error 상태로 메시지 표시 |
| 인증 만료 | refresh 실패 시 토큰 제거 후 로그인 화면으로 이동 |
| 비로그인 접근 | 라우터 가드 또는 버튼 문구를 통해 로그인 유도 |

## 4. 사용자 흐름

### 4.1 비로그인 사용자 흐름

1. 홈 화면에 접속한다.
2. 분석, 거래 검색, 공지사항, Q&A 목록/상세를 조회한다.
3. AI 리포트 생성, 마이페이지, Q&A 작성 등 로그인 필요 기능 접근 시 로그인 화면으로 이동한다.
4. 로그인 성공 후 원래 요청한 화면으로 redirect된다.

### 4.2 로그인 사용자 분석 흐름

1. `/analysis` 화면에 접속한다.
2. 이벤트와 분석 기간을 선택한다.
3. 지도와 패널에서 지역별 충격 전파 결과를 확인한다.
4. AI 리포트를 생성한다.
5. 리포트 미리보기 또는 PDF 다운로드를 수행한다.
6. 마이페이지에서 생성한 리포트 이력을 확인한다.

### 4.3 시나리오 확인 흐름

1. 분석 결과 또는 마이페이지에서 시나리오 생성/조회 흐름으로 진입한다.
2. `/scenarios/:scenarioId` 화면에서 라운드별 시장 반응을 확인한다.
3. 특정 라운드에 대해 AI 해설을 요청한다.
4. 지역별, 페르소나별 행동 설명을 확인한다.

### 4.4 게시판 흐름

1. 공지사항 또는 Q&A 목록을 조회한다.
2. 항목을 선택해 상세 화면으로 이동한다.
3. 로그인 사용자는 Q&A 작성, 댓글 작성, 본인 글 수정/삭제를 수행한다.
4. 관리자 또는 권한 보유 사용자는 공지 작성/수정/삭제를 수행한다.

| 화면 목적 | 분석 결과 기반 지역별 시장 반응 시나리오와 AI 해설 확인 |
| 주요 UI | 시나리오 요약, 라운드 탭, 지역별 시나리오 카드, 페르소나 상태, AI 해설 영역 |
| 사용자 액션 | 라운드 선택, 라운드 AI 해설 생성, 지역별 설명 확인 |
| 호출 API | `GET /api/scenarios/{scenarioId}`, `POST /api/scenarios/{scenarioId}/rounds/{relativeMonth}/explanation` |
| 입력 데이터 | scenarioId, relativeMonth |
| 출력 데이터 | 시나리오 문서, 라운드별 AI 해설, 지역/페르소나별 설명 |
| 예외 처리 | 시나리오 조회 실패 시 오류 상태, AI 해설 생성 중 버튼 비활성화 |
