# EstateFlow 기능 명세서

## 1. 문서 개요

본 문서는 EstateFlow 프로젝트의 주요 기능을 기능 그룹별로 분리하여 정리한 산출물이다.  
기존 `backend-fastAPI/기획서 수정 0515요약.md`의 기능 요구사항을 기반으로 하되, 개발을 하며 보완하였다.

## 2. 사용자 유형

| 사용자 유형 | 설명 | 주요 기능 |
| --- | --- | --- |
| 비회원 | 로그인하지 않은 사용자 | 회원가입, 로그인, 공지사항 조회 |
| 회원 | 로그인한 일반 사용자 | 마이페이지, Q&A 등록/조회, 분석 결과 조회, AI 보고서 생성 |
| 관리자 | 서비스 운영자 | 공지사항 관리, Q&A 답변, 분석 이벤트 및 데이터 관리 |

### 4.7 G7 시나리오 탐색

시나리오 탐색 기능은 분석 결과를 시장 참여자 관점으로 변환해 보여주는 기능이다.  
LLM이 시장을 임의로 예측하는 구조가 아니라, 분석 수치와 규칙 기반 상태 변화를 먼저 계산하고 LLM은 그 결과를 자연어로 설명하는 역할을 담당한다.

| ID | 기능 명세 | 구현 상태 | 비고 |
| --- | --- | --- | --- |
| G7-01 | 분석 결과 수치를 에이전트 초기 상태 파라미터로 변환한다. | 구현 | 지역별 가격·거래량·시차 기반 |
| G7-02 | 페르소나별 초기 상태를 설정하고 라운드를 시작한다. | 구현 | 실수요자, 투자자 등 |
| G7-03 | 규칙 기반으로 에이전트 행동과 시장 상태 변화를 계산한다. | 구현 | LLM 판단 배제 |
| G7-04 | 변경된 상태를 다음 라운드에 반영한다. | 구현 | 월 단위 흐름 |
| G7-05 | LLM API가 각 라운드의 상태 변화를 페르소나 언어로 설명한다. | 구현 | 자연어 해석 |
| G7-06 | 페르소나별 라운드 반응 흐름을 시각화한다. | 구현 | 시나리오 화면 |
| G7-07 | 라운드별 시장 분위기 변화를 출력한다. | 구현 | 시장 심리 설명 |
| G7-08 | 라운드별 지역 관심도 변화를 시각화한다. | 구현 | 관심 지역 변화 |
| G7-09 | 시나리오 결과 요약 보고서를 생성한다. | 구현 | AI 보고서 연계 |

## 8. 비기능 요구사항

| 구분 | 요구사항 |
| --- | --- |

| 성능 | 분석 결과 조회 시 사전 집계된 데이터를 활용해 응답 지연을 줄여야 한다. |

| 보안 | Q&A 등록, 마이페이지, 개인 보고서 조회 등 개인화 기능은 인증된 사용자만 접근할 수 있어야 한다. |
| 확장성 | 데이터 수집, 분석, 보고서, 시나리오 기능은 독립적으로 확장 가능해야 한다. |
